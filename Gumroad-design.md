# Гайд по проекту в Стиле Gumroad

Этот документ поможет вам понять структуру сайта, его дизайн-систему и как вносить изменения.

## 1. Обзор проекта

это лендинг для SaaS-платформы.

- **Технологии:** Сайт построен на чистом HTML, CSS и JavaScript (без фреймворков).
- **Цель:** Привлечь пользователей, собрать их email-адреса в список ожидания и предоставить юридическую информацию (Privacy Policy, Terms of Service).

## 2. Структура файлов

- `index.html`: Главная страница (лендинг).
- `privacy.html`: Страница с политикой конфиденциальности.
- `terms.html`: Страница с условиями использования.
- `main.js`: JavaScript-файл, содержащий всю логику сайта.
- `GUIDE.md`: Этот самый гайд.

## 3. Дизайн-система

Все основные стили определены в `<style>`-блоке внутри каждого HTML-файла. Они используют CSS-переменные для простоты и консистентности.

### Цветовая палитра

- `--c-bg: #F4F4F0` (Фон страницы)
- `--c-surface: #FFFFFF` (Фон для карточек и других элементов)
- `--c-ink: #0A0A0A` (Основной цвет текста)
- `--c-accent: #6D28D9` (Главный акцентный цвет - фиолетовый)
- `--c-accent-2: #A5B4FC` (Второстепенный акцентный цвет - светло-синий)
- `--c-warn: #FBBF24` (Цвет для предупреждений и ярких значков)

### Типографика

- **Основной шрифт:** `ui-sans-serif, system-ui, ...` (стандартный системный шрифт без засечек).
- **Заголовки:**
    - `<h1>`: Главный заголовок на странице.
    - `<h2>`: Заголовки секций.
    - `<h3>`: Подзаголовки внутри секций или на карточках.
- **Параграфы:** `<p>` для основного текста.

### Компоненты

- **Кнопки:**
    - `.button`: Базовый класс для всех кнопок.
    - `.button--primary`: Основная (фиолетовая) кнопка.
    - `.button--secondary`: Второстепенная (белая) кнопка.
    - `.button--large`: Увеличенная версия кнопки.
- **Карточки:**
    - `.card`: Базовый класс для всех карточек с тенью и скругленными углами.
- **Иконка сайта (Favicon):**
    - Иконка в формате SVG добавлена напрямую в HTML-код каждой страницы. Она использует основной акцентный цвет.

## 4. Ключевые скрипты и логика

Вся логика находится в файле `main.js`.

### Форма подписки (Waitlist)

- При отправке формы, email пользователя отправляется на вебхук `https://n8n.aaagency.at/...` методом `POST`.
- После успешной отправки форма скрывается и появляется сообщение "Thank you!".
- В случае ошибки появляется сообщение об ошибке.

### Баннер согласия (Cookie Consent)

- **Логика:** При первой загрузке сайта скрипт проверяет `localStorage` на наличие записи `consentStatus`.
- **Показ:** Если записи нет, внизу страницы появляется баннер с просьбой дать согласие.
- **Действия:**
    - **Accept:** Включается отслеживание для Google Analytics, в `localStorage` записывается `consentStatus: 'granted'`, баннер скрывается.
    - **Decline:** Отслеживание остается выключенным, в `localStorage` записывается `consentStatus: 'denied'`, баннер скрывается.
- **Настройки по умолчанию:** Сбор данных для аналитики **отключен** по умолчанию. Он включается только после согласия пользователя.

### Аналитика

- **Google Analytics** и **Meta Pixel** подключены в секции `<head>` на каждой HTML-странице. Их работа регулируется логикой баннера согласия.

## 5. Как вносить изменения

- **Изменение текста:** Просто откройте нужный HTML-файл (`index.html`, `privacy.html` или `terms.html`) и найдите нужный текстовый блок.
- **Добавление новой секции на главную:** Скопируйте одну из существующих секций (например, `<section class="section">...</section>`) в `index.html` и измените ее содержимое.
- **Изменение стилей:** Все стили находятся вверху каждого HTML-файла в теге `<style>`. Для глобальных изменений (например, цвета кнопок) достаточно поменять CSS-переменную в секции `:root`.
- **Важно:** Если вы добавляете новый скрипт аналитики или меняете иконку сайта (favicon), не забудьте внести эти изменения во **все три HTML-файла** для единообразия.
